A quick map of where to go for what.

| You want to…                              | Go to                                                          |
|-------------------------------------------|----------------------------------------------------------------|
| Ask a question, share an idea, or chat    | [GitHub Discussions](https://github.com/olafkfreund/TFactory/discussions) |
| Report a bug                              | [Bug report template](https://github.com/olafkfreund/TFactory/issues/new?template=bug_report.yml) |
| Suggest a feature or change in behaviour  | [Feature request via Discussions](https://github.com/olafkfreund/TFactory/discussions/categories/ideas) |
| Read the docs                             | [`README.md`](README.md), [`guides/`](guides/), [`CLAUDE.md`](CLAUDE.md) |
| Set up the project                        | [README → Quick Start](README.md#quick-start), [CONTRIBUTING.md](CONTRIBUTING.md#development-setup) |
| Understand release versions               | [RELEASE.md](RELEASE.md), [CHANGELOG.md](CHANGELOG.md)         |
| Contribute code                           | [CONTRIBUTING.md](CONTRIBUTING.md)                             |
| Report a security issue                   | [SECURITY.md](SECURITY.md) — **do not open a public issue**    |

## Before opening an issue

1. **Search existing issues + discussions first.** Most problems are already known.
2. **Check `CHANGELOG.md`** — your issue may be fixed in a newer release.
3. **Reproduce on the latest `main`** when possible.
4. Include: OS, browser (for UI bugs), exact TFactory version (`package.json` / `git rev-parse HEAD`), and the smallest steps to reproduce.

## Response times

This is a volunteer-maintained project. Issues are usually triaged
within a week. There is no commercial SLA.
